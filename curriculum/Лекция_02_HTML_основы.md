## Материал для лекции

### 1. История развития HTML

**Теория:**

HTML был создан Тимом Бернерс-Ли в 1991 году как язык для обмена научной информацией. Первая версия HTML включала базовые теги для заголовков, абзацев и ссылок. С развитием интернета HTML эволюционировал, добавляя новые возможности:

- **HTML 2.0 (1995):** стандартизация базовых тегов.
- **HTML 4.01 (1999):** поддержка таблиц, фреймов и стилей.
- **HTML5 (2014):** семантические теги, мультимедиа, новые API.

#### Исторические казусы и показательные кейсы

| Казус / период | Кратко | Пример / артефакт | Чем это закончилось / урок |
| -------------- | ------ | ----------------- | --------------------------- |
| "Tag Soup" (1990-е) | Страницы без строгой структуры: вложенные таблицы, отсутствие `DOCTYPE`. | Скриншоты старых порталов (поиск: "1998 web design"). | Появилась потребность в стандартизации, валидаторы W3C. |
| Browser Wars (Netscape vs IE) | Вендорские нестандартные теги (`<blink>`, `<marquee>`). | `<blink>` (Netscape), `<marquee>` (IE). | Осознание важности интероперабельности; стандарты W3C/WHATWG. |
| IE6 Dominance (2001–2006) | Зависимость от proprietary-функций ActiveX, отсутствие обновлений. | Старые корпоративные приложения под IE6. | Долг технической совместимости, торможение развития. |
| XHTML 2 vs HTML5 Расхождение | Попытка строгого XML-подхода против эволюции совместимости. | Дискуссии W3C vs WHATWG. | Победа прагматизма: HTML5 = эволюция, не разрыв. |
| Quirks Mode | Различный рендер из-за отсутствия правильного `<!DOCTYPE html>`. | Сравнение "standards" vs "quirks" рендеринга. | Всегда указываем современный DOCTYPE. |
| Табличная верстка для layout | Использование `<table>` для построения сеток. | Глубокие вложенные таблицы. | Переход к CSS layout (Flexbox / Grid). |
| Видео форматы (2008–2012) | Отсутствие единого формата: H.264 vs WebM vs Ogg. | `<video>` с множеством `<source>`. | Многоисточниковый подход, адаптация к браузеру. |
| Отсутствие доступности (a11y) | Игнорирование alt-текста, заголовков, структуры. | `<img src="decor.png">` без `alt`. | Включение WCAG, семантических тегов, aria-атрибутов. |
| Инлайновые стили и шрифты | `<font color="red">` в контенте. | Спагетти из `<font>` / `style="..."`. | Отделение структуры (HTML) и представления (CSS). |
| Всплывающие окна и злоупотребление JS | `window.open` для рекламы. | Pop-up спам. | Блокировщики, лучшая UX-практика. |

##### Подробные объяснения (понятно для начинающих)

**1. "Tag Soup" — хаотичная разметка**  
До появления строгих рекомендаций разработчики часто писали HTML без закрывающих тегов или с неправильной вложенностью. Браузеры пытались "догадаться" что имелось в виду.

Плохой пример:

```html
<body>
  <h1>Новости
  <p>Сегодня произошло...</p>
  <ul>
    <li>Пункт 1
    <li>Пункт 2
</body>
```

Как исправить:

```html
<body>
  <h1>Новости</h1>
  <p>Сегодня произошло...</p>
  <ul>
    <li>Пункт 1</li>
    <li>Пункт 2</li>
  </ul>
</body>
```

Совет: всегда проверяйте закрытие тегов и вложенность.

**2. Browser Wars — нестандартные теги**  
Браузеры соревновались, добавляя свои "фишки". Например:

```html
<blink>Мигающий текст (Netscape)</blink>
<marquee>Бегущая строка (IE)</marquee>
```

Проблема: в других браузерах эффект не работал. Сейчас вместо этого используем CSS-анимации:

```html
<p class="blink">Акцент</p>
```

```css
.blink { animation: pulse 1s infinite alternate; }
@keyframes pulse { from { opacity: .2; } to { opacity: 1; } }
```

**3. Доминирование IE6**  
Компании писали сайты только "под IE6" (используя устаревшие технологии). Такие сайты плохо работают сегодня. Урок: не завязывайтесь на одного поставщика — тестируйте в нескольких браузерах.

**4. XHTML 2 против HTML5**  
XHTML 2 требовал строгого XML: малейшая ошибка = страница не работает. HTML5 выбрал стратегию: "работать даже при ошибках" + постепенно вводить новые семантические элементы.

**5. Quirks Mode**  
Если не указать современный DOCTYPE, браузер переходит в режим совместимости и отображает страницу по старым правилам.

Минимально правильно:

```html
<!DOCTYPE html>
<html lang="ru">
<head><meta charset="UTF-8"><title>OK</title></head>
<body>...</body>
</html>
```

**6. Табличная верстка**  
Раньше таблицы использовали для позиционирования:

```html
<table width="100%">
  <tr>
    <td width="200">Меню</td>
    <td>Контент</td>
  </tr>
</table>
```

Теперь используем Flexbox:

```html
<div class="layout">
  <nav>Меню</nav>
  <main>Контент</main>
</div>
```

```css
.layout { display: flex; }
nav { width: 200px; }
main { flex: 1; }
```

**7. Видео форматы**  
Чтобы видео работало в разных браузерах, добавляли несколько источников:

```html
<video controls>
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  Ваш браузер не поддерживает видео.
</video>
```

**8. Доступность (Accessibility / a11y)**  
Отсутствие `alt` делало контент невидимым для людей с нарушениями зрения.

```html
<img src="chart.png" alt="Гистограмма продаж за 2024 год">
```

Совет: alt должен описывать смысл, а не файл: НЕ "image1.png".

**9. Инлайновые стили и `<font>`**  
Плохой подход:

```html
<p><font color="red" size="5">Ошибка!</font></p>
```

Лучше:

```html
<p class="error">Ошибка!</p>
```

```css
.error { color: #d00000; font-size: 1.5rem; }
```

**10. Pop-up злоупотребления**  
Раньше сайты открывали множество всплывающих окон:

```html
<script>
  for (let i = 0; i < 5; i++) {
    window.open('ad.html');
  }
</script>
```

Сейчас такой подход блокируется и вызывает раздражение. Используйте ненавязчивые модальные окна или уведомления.

**Итого:** каждый казус показывает: стандарты и аккуратность = долговечность сайта.

**Нестандартные теги, ушедшие в историю:** `blink`, `marquee`, `font`, `center`, `frame`, `frameset`, `big`, `tt`.

**Уроки эволюции:**

1. Совместимость важнее идеологической чистоты (пример: отказ от XHTML 2).
2. Семантика и доступность = устойчивость контента (поисковые системы + assistive tech).
3. Минимум вендор-специфичных расширений = меньше технического долга.
4. Явный DOCTYPE и валидная структура предотвращают скрытые баги отображения.
5. Отделение структуры, стилей и поведения (HTML + CSS + JS) упрощает сопровождение.

### 2. Что такое HTML

**Теория:**

HTML (HyperText Markup Language) — язык разметки для создания веб-страниц. Описывает структуру и содержимое документа.

---

### 3. Базовая структура HTML-документа

**Теория:**

- HTML-документ начинается с `<!DOCTYPE html>`.
- Основные теги: `<html>`, `<head>`, `<body>`.
- В `<head>` размещаются метаданные: `<title>`, `<meta>`, `<link>`, `<script>`.
- В `<body>` — содержимое страницы: текст, изображения, формы, навигация.

**Пример:**

```html
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <title>Пример HTML</title>
  </head>
  <body>
    <h1>Заголовок</h1>
    <p>Это абзац текста.</p>
    <ul>
      <li>Пункт 1</li>
      <li>Пункт 2</li>
    </ul>
    <a href="https://example.com">Ссылка</a>
    <img src="https://via.placeholder.com/150" alt="Пример изображения">
  </body>
</html>
```

---

### 4. Основные теги

**Теория:**

- `<h1>`–`<h6>` — заголовки.
- `<p>` — абзац.
- `<ul>`, `<ol>`, `<li>` — списки.
- `<a href="...">` — ссылка.
- `<img src="..." alt="...">` — изображение.

---

### 5. Семантические теги HTML5

**Теория:**

- Семантические теги делают структуру страницы понятной для браузеров, поисковиков и людей.
- Основные теги: `<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<aside>`, `<footer>`.

**Пример:**

```html
<header>
  <h1>Мой блог</h1>
  <nav>
    <a href="/">Главная</a>
    <a href="/about">О сайте</a>
  </nav>
</header>
<main>
  <article>
    <h2>Первая запись</h2>
    <p>Текст статьи...</p>
  </article>
  <aside>
    <h3>Сайдбар</h3>
    <p>Полезные ссылки</p>
  </aside>
</main>
<footer>
  &copy; 2024 Мой блог
</footer>
```

### 6. Формы, медиа, новые возможности

**Теория:**

- Формы: `<form>`, `<input>`, `<textarea>`, `<button>`, `<select>`, `<option>`.
- Новые типы `<input>`: email, url, date, range, color, number.
- Новые атрибуты: `placeholder`, `required`, `autofocus`, `pattern`.
- Медиа: `<img>`, `<audio>`, `<video>`.

**Пример формы:**

```html
<form>
  <input type="email" placeholder="Ваш email" required>
  <input type="date">
  <button type="submit">Отправить</button>
</form>
```

**Пример видео:**

```html
<video controls width="320">
  <source src="https://www.example.com/movie.mp4" type="video/mp4">
  Ваш браузер не поддерживает видео.
</video>
```

- Вставьте видео и аудио с помощью тегов `<video>` и `<audio>`.
