## Описание

Реализация интерактивных компонентов на чистом JS, обработка событий (включая делегирование), доступность с клавиатуры и клиентская валидация формы.

## Цели

- Освоить работу с DOM: добавление, удаление и изменение элементов интерфейса.
- Научиться обрабатывать события, использовать делегирование, работать с клавиатурной доступностью.
- Реализовать дружелюбную клиентскую валидацию форм.

## Основные возможности

- Аккордеон для списка книг с ARIA-поддержкой
- Табы: "Добавить книгу" и "Статистика"
- Модальное окно для деталей книги с focus-trap
- Форма с валидацией (обязательные поля, email, minlength)
- Делегирование событий для списка книг (лайк, смена статуса, удаление, детали)
- Сохранение/загрузка данных из `localStorage`
- Поддержка тёмной темы через CSS-переменные и `prefers-color-scheme`
- Адаптивная вёрстка (mobile-first)

## Структура проекта

- `index.html` — основная страница (семантическая разметка)
- `styles.css` — все стили, переменные тем и медиазапросы
- `scripts/main.js` — точка входа, инициализация модулей
- `scripts/components/` — модульные компоненты:
  - `accordion.js`, `tabs.js`, `modal.js`, `form.js`, `bookList.js`, `theme.js`
- `assets/` — изображения и ресурсы

## Как запустить проект (быстро)

Проект — статический, поэтому достаточно открыть `index.html` в браузере. Ниже — несколько удобных способов для Windows PowerShell.

1) Открыть файл прямо (быстро)

   - В Проводнике — дважды клик по `index.html`.
   - Через PowerShell (в каталоге `lab_2`):

```powershell
Set-Location -LiteralPath "D:\Предметы и литература\Веб-технологии\WEB-2025\lab_2"
ii .\index.html
```

1) Локальный HTTP-сервер с Python (рекомендуется для корректной работы относительных URL и CORS)

   - Если у вас установлен Python 3:

```powershell
Set-Location -LiteralPath "D:\Предметы и литература\Веб-технологии\WEB-2025\lab_2"
# либо
py -3 -m http.server 5001
# или
python -m http.server 5001
```

- Откройте в браузере: `http://localhost:5001`

1) Через VS Code + Live Server (удобно при разработке)

   - Установите расширение *Live Server* в VS Code.
   - Откройте папку `lab_2` и нажмите "Go Live" или правой кнопкой по `index.html` → "Open with Live Server".

1) Node.js вариант (если установлен `http-server`):

```powershell
Set-Location -LiteralPath "D:\Предметы и литература\Веб-технологии\WEB-2025\lab_2"
# если не установлен, выполнить один раз:
npm install -g http-server
http-server -p 5001
```

## Короткие примечания

- Если страницы не загружаются корректно при открытии файла, используйте локальный сервер (Python/Live Server).  
- Путь к проекту в командах учитывает пробелы и кириллицу — используйте `Set-Location -LiteralPath "<путь>"` в PowerShell.

## Как пользоваться

- Откройте вкладку "Добавить книгу" и заполните форму (название и автор — обязательно).  
- Кнопка "Добавить книгу" активна только при валидной форме.  
- После добавления карточки книги можно ставить лайк, менять статус (для чтения/прочитано), смотреть детали в модальном окне и удалять запись.  
- Все изменения сохраняются в `localStorage` — при перезагрузке список восстановится.

## Доступность и тестирование

- Проект учитывает базовые требования a11y: ARIA-атрибуты, логичная табуляция, видимая фокусировка, фильмы для модального окна.
- Рекомендуется проверять страницу в Lighthouse (Accessibility, Best Practices) и валидаторе HTML/CSS.
