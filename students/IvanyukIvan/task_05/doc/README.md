# REST API для управления учебными группами и домашними заданиями

Вариант - 32.
Выполнил - Иванюк Иван.

RESTful API для хранилища файлов с правами доступа и версиями с валидацией, обработкой ошибок и документацией Swagger.

## Запуск проекта

### Требования

* Node.js 16+
* npm

### Установка

npm install

### Запуск в режиме разработки

npm run dev

### Запуск в production

npm start

### Переменные окружения

Создайте файл .env в корне проекта:

PORT=3000
NODE_ENV=development

## API Endpoints

### Files Routes (/api/groups)

* POST /api/v1/files - Загрузить файл
* GET /api/v1/files - Получить список файлов
* GET /api/v1/files/:id - Получить информацию о файле
* GET /api/v1/files/:id/download - Скачать файл
* PATCH /api/v1/files/:id - Обновить метаданные
* DELETE /api/v1/files/:id - Удалить файл

### Health Check

* GET /health - проверка работоспособности сервера

## Документация API

* Документация Swagger доступна по адресу: /docs
* Валидация данных
* Группы (Files Routes)

* file: Файл (обязательно, до 10MB)
* name: строка, 1-255 символов (обязательно)
* description: строка, до 1000 символов (опционально)
* tags: JSON массив строк, до 50 символов каждая (опционально)
* isPublic: boolean (опционально, по умолчанию false)

## Коды ответов

* 200: Успешный запрос
* 201: Ресурс создан
* 204: Ресурс удален
* 400: Неверный запрос
* 401: Не авторизован
* 403: Запрещено (нет прав)
* 404: Не найдено
* 409: Конфликт
* 422: Ошибка валидации
* 500: Внутренняя ошибка сервера

## Хранение данных

Данные хранятся в памяти (in-memory storage) и сбрасываются при перезапуске сервера.
