## Ход выполнения работы

### 1. Структура проекта

✍️ Структура упрощена и разделена на папки `doc/` и `src/`.

- doc/README.md — отчёт по лабораторной работе
- doc/screenshots/ — скриншоты Lighthouse
- src/public/index.html — главная страница каталога стартапов
- src/public/style.css — стили
- src/public/script.js — простая логика отображения
- src/lib/catalog.js — мок-данные каталога
- src/tests/unit/basic.test.js — юнит-тесты
- src/tests/integration/dom.test.js — интеграционный тест на jsdom
- src/Dockerfile — контейнер для запуска статики
- src/docker-compose.yml — запуск контейнера
- src/.github/workflows/ci.yml — базовый pipeline
- src/package.json — скрипты и зависимости
- src/jest.config.js — конфигурация jest

### 2. Реализованные элементы

- Разметка каталога стартапов с карточками
- Фильтрация проектов по названию
- Стили для отображения карточек
- Мок-данные в отдельном модуле
- Юнит и интеграционные тесты на Jest + jsdom
- Многостадийный Dockerfile и docker-compose
- Workflow GitHub Actions (install → lint → test → build)
- Скриншоты Lighthouse из открытого API
