## Цель работы

Целью лабораторной работы является изучение методов повышения качества веб‑приложений, включая автоматизированное тестирование, подготовку production‑сборки, контейнеризацию с использованием Docker, а также конфигурирование проекта для дальнейшего деплоя и CI/CD.

---

## Вариант №39

Система управления турнирами и матчами.

---

## Краткое описание проекта

В качестве основы используется SPA‑приложение, разработанное в лабораторной работе №7.  
Приложение реализовано с применением следующих технологий:

- React + TypeScript
- Vite
- Redux Toolkit + RTK Query
- React Router
- json‑server (mock REST API)

---

## Ход выполнения работы

### 1. Подготовка тестового окружения

Для тестирования проекта был установлен и настроен инструмент **Vitest** совместно с **React Testing Library**.

Установленные зависимости:
- vitest
- jsdom
- @testing-library/react
- @testing-library/jest-dom

Был создан `vitest.config.ts` для изоляции тестовой среды от production.

---

### 2. Реализация тестов

Реализован integration‑тест для асинхронного компонента списка турниров.  
Проверяемый сценарий: корректное отображение индикатора загрузки при выполнении запроса данных через RTK Query.

Тест выполняется командой:

```
npm test
```

---

### 3. Подготовка production‑сборки

Для корректной сборки проекта были выполнены следующие действия:
- тестовые файлы исключены из production‑сборки TypeScript;
- конфигурации Vite и Vitest разделены;
- успешная сборка проекта выполняется командой:

```
npm run build
```

В результате формируется директория `dist`.

---

### 4. Контейнеризация приложения

Для контейнеризации приложения был создан `Dockerfile` с multi‑stage сборкой:
- на первом этапе выполняется сборка React‑приложения;
- на втором этапе production‑версия запускается через Nginx.

Сборка Docker‑образа:

```
docker build -t tournaments-app .
```

Запуск контейнера:

```
docker run -p 8080:80 tournaments-app
```

Приложение доступно по адресу `http://localhost:8080`.

---

### 5. Оценка качества приложения

Для оценки использовался **Lighthouse**. Проверены показатели:
- Performance
- Accessibility
- Best Practices
- SEO

Результаты соответствуют рекомендуемым значениям.
