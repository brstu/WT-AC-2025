## Цель работы

Целью лабораторной работы является изучение методов обеспечения качества SPA‑приложения интернет‑магазина: автоматизированное тестирование компонентов и интеграций, подготовка production‑сборки, контейнеризация с использованием Docker и подготовка к деплою.

---

## Вариант №10

Тестирование и деплой мини‑магазина (SPA).

---

## Краткое описание проекта

В качестве основы используется SPA‑приложение, переработанное под мини‑магазин.  
Приложение реализовано с применением следующих технологий:

- React + TypeScript
- Vite
- Redux Toolkit + RTK Query
- React Router
- json‑server (mock REST API)
- Vitest + React Testing Library
- Docker для контейнеризации

---

## Ход выполнения работы

### 1. Подготовка тестового окружения

Для тестирования проекта использованы **Vitest** и **React Testing Library**. Конфигурация тестовой среды отделена от production‑сборки через `vitest.config.ts`.

Установленные зависимости:

- vitest
- jsdom
- @testing-library/react
- @testing-library/jest-dom

---

### 2. Реализация тестов

Реализованы unit и integration тесты для основных UI‑компонентов магазина (список товаров, индикатор загрузки, обработка ошибок). Тесты запускаются командой:

```bash
npm test
```

---

### 3. Подготовка production‑сборки

Сборка выполняется командой:

```bash
npm run build
```

В результате формируется директория `dist`, готовая к деплоя.

---

### 4. Контейнеризация приложения

Создан `Dockerfile` с multi‑stage сборкой: на этапе сборки формируется production‑бандл, во втором этапе используется Nginx для отдачи статики.

Сборка Docker‑образа:

```bash
docker build -t mini-shop .
```

Запуск контейнера:

```bash
docker run -p 8080:80 mini-shop
```
