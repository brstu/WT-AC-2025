## Структура проекта

- index.html — основная разметка интерактивного гида.
- styles.css — стили, включая адаптивность и поддержку тёмной темы.
- script.js — основная логика работы компонентов, модальных окон, табов, аккордеона и избранного.
- validation.js — модуль с функцией getValidationMessage для клиентской валидации формы.
- tests.js — файл с юнит-тестами для функции валидации.
- readme.md — отчёт о лабораторной работе (этот файл).
- Папка img/ — изображения достопримечательностей (используются в модальных окнах).

## Реализованные элементы

- Семантическая разметка HTML5 (header, main, section, article и т. д.).
- Адаптивный макет с двумя основными брейкпоинтами (для мобильных и десктопов).
- Интерактивные табы для переключения районов.
- Аккордеон для скрытия/отображения описания мест.
- Модальные окна для детального просмотра достопримечательностей.
- Функция «Избранное» с сохранением данных в localStorage.
- Форма обратной связи с кастомной клиентской валидацией.
- Юнит-тесты для проверки логики валидации (tests.js).
- Тёмная тема с поддержкой prefers-color-scheme и ручным переключением.

## Архитектура вёрстки

### Медиазапросы

- @media (max-width: 768px): Используется для мобильной версии (гамбургер-меню, вертикальное расположение табов, адаптация карточек).

### Тёмная тема

- Реализована с использованием CSS-переменных (:root) для управления цветовой палитрой.
- Автоматическое включение: Поддержка системной тёмной темы через @media (prefers-color-scheme: dark).
- Ручное переключение: Кнопка Темная тема сохраняет выбор пользователя в localStorage.

### Доступность (Accessibility)

* **ARIA-атрибуты**: Широко используются для всех интерактивных компонентов:
    * Табы: `role="tablist"`, `aria-selected`.
    * Кнопки избранного: `aria-pressed`.
    * Форма: `aria-live="polite"` для сообщений об ошибках.
* **Фокус**: Реализовано управление фокусом для интерактивных элементов и модальных окон (включая **возврат фокуса** при закрытии).
