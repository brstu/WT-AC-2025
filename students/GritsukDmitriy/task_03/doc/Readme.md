# Отчёт по лабораторной работе 03: Каталог игр

## Реализованные функции

1. **Загрузка данных с API RAWG**:
   - Отображение 12 игр на странице.
   - Поиск по названию и фильтр по жанру.
   - Пагинация с динамическим расчётом страниц.

2. **Асинхронность и обработка ошибок**:
   - Использование `async/await` с `try/catch/finally`.
   - Индикаторы загрузки, ошибок, пустого состояния.
   - Скелетоны во время загрузки.

3. **Ретраи и таймауты**:
   - Функция `fetchWithRetry` с экспоненциальной задержкой.
   - 3 попытки при ошибках сети/сервера.
   - Таймаут 5 секунд на запрос.

4. **Отмена запросов**:
   - `AbortController` для отмены предыдущих запросов при новом поиске.
   - Комбинированный сигнал с таймаутом.

5. **Кэширование**:
   - In-memory кэш на `Map` с TTL 30 секунд.
   - Ключ: `запрос-жанр-страница`.
   - Индикатор "Из кэша".

6. **UX-улучшения**:
   - Кнопка "Обновить" с игнорированием кэша.
   - Дебаунс ввода (500 мс).
   - Плавные анимации и адаптивный дизайн.

## Описание работы ретраев/таймаутов/отмены

### Ретраи:
- При ошибке сети или сервера (>399) выполняется повторная попытка.
- Экспоненциальная задержка: 1с, 2с, 3с.
- После 3 неудачных попыток — ошибка пользователю.

### Таймауты:
- Каждый запрос ограничен 5 секундами.
- Реализован через `AbortController` и `setTimeout`.
- При превышении — `AbortError`.

### Отмена:
- При новом поиске или смене страницы предыдущий запрос отменяется.
- Используется `AbortController.abort()`.
- Позволяет избежать "гонки запросов".

## Запуск локально
1. Клонировать репозиторий.
2. Получить API ключ на [RAWG.io](https://rawg.io/apidocs).
3. Вставить ключ в `main.js` в переменную `API_KEY`.
4. Открыть `index.html` в браузере или запустить через Live Server.
