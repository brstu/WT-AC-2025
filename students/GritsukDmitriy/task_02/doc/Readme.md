# Лабораторная работа 02. Трекер привычек

— Студент: Грицук Дмитрий

## Цели

- Работа с DOM и событиями (в т.ч. делегирование)
- Реализация доступных интерактивных компонентов с клавиатурной навигацией
- Клиентская валидация формы с дружелюбными сообщениями
- Сохранение состояния интерфейса (localStorage)

## Реализованные компоненты

- Аккордеон раздела FAQ (часто задаваемые вопросы о привычках)
- Модальное окно для подтверждения удаления привычек (с фокус-ловушкой)
- Форма добавления привычки с валидацией (название, категория, цель, сложность)
- Трекер привычек: лайк/удаление/редактирование через делегирование событий
- Переключатель темы (светлая/тёмная, поддержка системной темы)
- Панель статистики с подсчётом привычек и лайков

Файлы см. в `src/` (`index.html`, `styles.css`, `scripts/*.js`).

## Как проверить

1) Откройте проект и запустите локальный сервер из папки `src` (например, через расширение Live Server в VS Code).
2) Проверьте:
   - Аккордеон: Клики по заголовкам аккордеона, Управление с клавиатуры (Enter/Space для открытия/закрытия), Сохранение состояния между перезагрузками
   - Модальное окно: Кнопка "Удалить" открывает модальное окно подтверждения, Закрытие через: Esc, клик по фону, крестик, Фокус возвращается на кнопку удаления после закрытия
   - Форма добавления привычки: Валидация в реальном времени (при вводе и при потере фокуса), Пользовательские сообщения об ошибках под каждым полем, Кнопка "Добавить привычку" активна только при корректно заполненной форме, Успешная отправка показывает сообщение и сбрасывает форму
   - Карточки привычек: Лайк: меняет счётчик и сохраняет состояние в localStorage, Удаление: открывает модальное окно подтверждения, Редактирование: переносит данные привычки в форму для изменения
   - Переключатель темы: Ручное переключение светлой/тёмной темы, Автоматическое определение системной темы

Примечание: для корректной загрузки модулей JS необходим запуск через веб‑сервер (не просто открывать файл).

## Доступность (a11y)

- Клавиатурная поддержка: Tab/Shift+Tab, Enter/Space, Esc; стрелки/Home/End в аккордеоне
- Проверка ARIA-атрибутов в инструментах разработчика
- Использование скринридера для проверки семантики
- Фокус-менеджмент: trap в модалке, возврат фокуса по закрытию
- Визуальная доступность: заметные стили фокуса, достаточный контраст, поддержка `prefers-reduced-motion`
