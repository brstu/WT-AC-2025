## Вариант №17 — “Тестирование и деплой мероприятий”

---

## Ход выполнения работы

### Цель работы

Настроить качество веб-проекта для управления мероприятиями: реализовать тестирование (unit, integration, e2e), контейнеризацию с использованием Docker, базовый CI/CD pipeline в GitHub Actions и провести оценку качества интерфейса с помощью Lighthouse.

---

### Настройка и запуск

```bash
npm install
npm run dev
```

---

### Запуск тестов

```bash
npm test
npm run test:e2e
npm run test:coverage
```

---

### Запуск приложения в Docker

```bash
docker-compose up --build
docker run -p 3000:3000 event-app
```

---

### Проверка качества

```bash
npm start &
npm run lighthouse
```

---

### Вывод

В ходе выполнения лабораторной работы была успешно настроена система управления качеством для веб-приложения мероприятий. Реализованы:
* Комплексное тестирование на трех уровнях (unit, integration, e2e)
* Контейнеризация с помощью Docker и оптимизированный production образ
* Полный CI/CD pipeline в GitHub Actions с автоматическим деплоем
* Контроль качества интерфейса через Lighthouse с высокими показателями

Все требования варианта №17 выполнены в полном объеме. Проект готов к промышленному развертыванию и соответствует современным стандартам веб-разработки.

Приложение демонстрирует стабильную работу, высокую производительность и хорошую сопровождаемость благодаря настроенным процессам контроля качества.
