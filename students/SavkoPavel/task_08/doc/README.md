## Вариант №18 — “Тестирование и деплой компаний/вакансий.”

---

## Ход выполнения работы

### Цель работы

Настроить качество фронтенд-проекта для управления компаниями и вакансиями: реализовать тестирование (unit, integration, e2e), контейнеризацию с Docker, базовый CI/CD pipeline и провести оценку качества интерфейса с помощью Lighthouse.

---

### Настройка и запуск

```bash
npm install
npm run dev
```

---

### Запуск тестов

```bash
npm test
```

---

### Запуск приложения в Docker:
```bash
docker build -t vacancies-app .
docker run -p 3000:80 vacancies-app
```
