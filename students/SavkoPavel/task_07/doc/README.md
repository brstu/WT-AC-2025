## Ход выполнения работы

## Цель работы

Целью лабораторной работы является разработка серверного REST API на платформе Node.js с использованием фреймворка Express. В рамках работы необходимо реализовать CRUD-эндпоинты для выбранного ресурса, добавить валидацию входных данных, централизованную обработку ошибок и автоматическую документацию API с помощью Swagger (OpenAPI).

---

## Описание проекта

Целью лабораторной работы является разработка SPA (Single Page Application) на платформе React с использованием инструментов современного фронтенда.

---

## Архитектура проекта

### Структура

- **src/components** ― переиспользуемые компоненты интерфейса (Layout, Navbar и др.)\
- **src/page** ― страницы приложения (Home, CompaniesList, CompanyDetail, CompanyForm, NotFound)\
- **src/features/companies** ― логика работы с компаниями через RTK Query: endpoints, хуки, CRUD\
- **src/app/store.js** ― Redux store с подключением RTK Query\
- **src/router/router.jsx** ― маршруты приложения с использованием createHashRouter и вложенных маршрутов\
- **vite.config.js** ― конфигурация Vite\

---

## Реализованный функционал (CRUD)

| Метод | Эндпоинт | Описание |
|------------------------------------------------------|-----------|
| GET | /companies | Получение списка компаний |
| GET | /companies/{id} | Получение компаний по ID |
| POST | /companies | Создание новой компании |
| PUT | /companies/{id} | Обновление компании |
| DELETE | /companies/{id} | Удаление компании |

---

## Используемые технологии

1. React 18
2. Vite (билд и сборка проекта)
3. React Router v6 (маршрутизация, вложенные и защищённые маршруты)
4. Redux Toolkit + RTK Query (состояние, работа с API, кэширование)
5. React Hook Form (валидация форм)

---

## Итоги

В ходе выполнения лабораторной работы был разработан полноценный SPA на React с корректной архитектурой, управлением состоянием через Redux Toolkit + RTK Query, формами с валидацией и полноценным CRUD через публичный API.
