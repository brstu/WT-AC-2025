## Ход выполнения работы

## Описание проекта

Приложение — каталог книг с полноценным CRUD-функционалом, реализованное как SPA без использования фреймворков. Клиент работает с локальным хранилищем (localStorage) как с API.
Реализованы экраны:

- `Список книг с поиском`
- `Детальная информация о книге`
- `Форма добавления новой книги`
- `Форма редактирования существующей книги`

---

## Архитектура клиента

### Модульная структура

- **router.js** ― hash-маршрутизатор на основе событий hashchange/load\
- **api.js** ― модуль для работы с данными (CRUD операции)\
- **views.js** ― модуль представлений (экраны приложения)\
- **app.js** ― основной модуль приложения\

---

## UX-улучшения

- Подтверждение удаления через модальное окно\
- Блокировка формы во время отправки данных\
- Валидация полей формы\

## Сценарии тестирования

1. Просмотр списка книг — переход по #/books\
2. Поиск книг — ввод запроса в поле поиска\
3. Просмотр деталей — клик по кнопке "Подробнее"\
4. Добавление книги — переход по #/new и заполнение формы\
5. Редактирование книги — клик по иконке редактирования\
6. Удаление книги — клик по кнопке удаления с подтверждением
