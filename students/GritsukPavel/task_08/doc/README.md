## Ход выполнения работы

### 1. Структура проекта

```
src/
├── index.html              # Главная страница портфолио
├── style.css              # Стили приложения
├── script.js              # Основная логика
├── script.test.js         # Unit тесты
├── package.json           # Зависимости проекта
├── jest.config.json       # Конфигурация тестов
├── Dockerfile             # Docker образ
├── docker-compose.yml     # Docker Compose конфигурация
└── .github/
    └── workflows/
        └── ci.yml         # CI/CD pipeline

doc/
├── README.md              # Документация
└── screenshots/           # Скриншоты
    ├── lighthouse.png
    ├── tests.png
    └── docker.png
```

### 2. Реализованные элементы

**Тестирование:**

- Настроен Jest для unit-тестирования
- Реализовано 3 unit теста для базовых функций
- Тесты проверяют математические операции и функции работы с проектами

**Контейнеризация:**

- Создан Dockerfile для сборки образа приложения
- Настроен docker-compose.yml для локального запуска
- Приложение запускается на порту 8080

**CI/CD:**

- Настроен GitHub Actions workflow
- Pipeline включает этапы: установка зависимостей, линтинг, тесты, сборка
- Автоматический запуск при push и pull request в main

**Функционал приложения:**

- Отображение портфолио проектов
- Фильтрация проектов по категориям (все, веб, мобильные)
- Форма добавления новых проектов
- Использование изображений из внешнего API

## Инструкции по запуску

### Локальный запуск

```bash
cd src
npm install
npm start
```

Приложение будет доступно по адресу: <http://localhost:8080>

### Запуск тестов

```bash
cd src
npm test
```

### Запуск через Docker

```bash
cd src
docker-compose up
```

### Сборка Docker образа

```bash
cd src
docker build -t portfolio-app .
docker run -p 8080:8080 portfolio-app
```
