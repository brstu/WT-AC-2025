## Ход выполнения работы

### 1. Структура проекта

Проект представляет собой веб-каталог художественных работ с возможностью фильтрации по типам.

**Структура файлов:**

- `index.html` — основная страница каталога
- `app.js` — логика приложения и данные
- `server.js` — Express сервер для запуска приложения
- `app.test.js` — тесты для проверки функционала
- `package.json` — конфигурация проекта и зависимости
- `Dockerfile` — контейнеризация приложения
- `docker-compose.yml` — оркестрация контейнеров
- `.github/workflows/ci.yml` — CI/CD pipeline

### 2. Реализованные элементы

**Основной функционал:**

- Каталог художественных работ с изображениями
- Фильтрация по типам: живопись, скульптура, графика
- Отображение информации о работах (автор, год, тип)

**Тестирование:**

- Unit тесты для функций sum, getTypeLabel, calculateTotal
- Integration тест для getItemById
- Конфигурация Jest

**Контейнеризация:**

- Dockerfile для сборки образа приложения
- docker-compose для локального запуска

**CI/CD:**

- GitHub Actions workflow
- Установка зависимостей и запуск тестов

## Инструкции по запуску

### Локальный запуск

```bash
cd src
npm install
npm start
```

Приложение будет доступно по адресу: <http://localhost:3000>

### Запуск тестов

```bash
cd src
npm test
```

### Запуск через Docker

```bash
cd src
docker-compose up
```

### CI/CD

Pipeline запускается автоматически при push в ветку main. Включает установку зависимостей и запуск тестов.
