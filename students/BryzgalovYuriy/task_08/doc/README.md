## Ход выполнения работы

### 1. Структура проекта

```
task_08/
├── doc/
│   └── README.md
│   
└── src/
    ├── __tests__/
    │   ├── App.test.jsx
    │   ├── List.test.jsx
    │   └── Create.test.jsx
    ├── test/
    │   └── setup.js
    ├── .github/
    │   └── workflows/
    │       └── ci.yml
    ├── Dockerfile
    ├── docker-compose.yml
    ├── vitest.config.js
    ├── index.html
    ├── package.json
    └── src/
        ├── App.jsx
        ├── pages/
        └── main.jsx
```

**Основные файлы:**

  App.jsx — корневой компонент React-приложения
  pages/ — страницы списка, создания и просмотра стартапов
  __tests__/ — unit-тесты компонентов
  vitest.config.js — конфигурация тестового окружения
  Dockerfile — контейнеризация приложения
  docker-compose.yml — упрощённый запуск контейнера
  .github/workflows/ci.yml — CI pipeline GitHub Actions

### 2. Реализованные элементы

**Приложение:**

  SPA на React (Vite)
  Справочник стартапов
  Страницы:
  список стартапов
  добавление нового стартапа
  навигация и базовая маршрутизация
  Используется TanStack Query для работы с данными

**Тестирование:**

  Использован Vitest и @testing-library/react
  Реализованы unit-тесты компонентов:
  App.test.jsx — проверка рендера шапки и навигации
  List.test.jsx — проверка отображения списка
  Create.test.jsx — проверка отправки формы

**Docker:**

  Реализован многостадийный Dockerfile
  Приложение собирается и запускается в контейнере
  Добавлен docker-compose.yml для локального запуска

**CI/CD:**

  Настроен GitHub Actions
  Pipeline автоматически запускается при push и pull request
  Этапы CI:
    Установка зависимостей
    Запуск тестов
    Сборка проекта

**Качество:**

- Приложение доступно для проверки через Lighthouse

## Инструкции по запуску

### Локальный запуск

1. Открыть `index.html` в браузере
2. Или запустить локальный сервер:

```bash
cd src
python -m http.server 8000
```

### Запуск через Docker

```bash
cd src
docker-compose up
```

Приложение будет доступно по адресу: `http://localhost:8080`

### Запуск тестов

```bash
cd src
npm install
npm test
```

## Вывод

В ходе выполнения лабораторной работы был создан проект "Приют для питомцев" с применением современных практик разработки. Были реализованы unit тесты с использованием Jest, настроена контейнеризация через Docker и Docker Compose, создан CI pipeline в GitHub Actions для автоматической проверки кода и запуска тестов.

Приложение использует публичные API для получения изображений питомцев, что позволяет динамически отображать контент. Реализован базовый функционал поиска и взаимодействия с питомцами.

В процессе работы были освоены следующие навыки:

- Написание и настройка unit тестов
- Работа с Docker и Docker Compose
- Настройка CI/CD pipeline в GitHub Actions
- Проверка качества веб-приложения через Lighthouse

Использованные инструменты: HTML, CSS, JavaScript, Jest, Docker, GitHub Actions, Nginx, Dog API, Cat API.
