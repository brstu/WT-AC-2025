## Цель работы

Создание серверного REST API на платформе Node.js с использованием фреймворка Express. Реализация базовых CRUD-операций, валидации входных данных, обработки ошибок и документации API с помощью Swagger UI.

### Вариант №26

## Ход выполнения работы

### 1. Структура проекта

В рамках лабораторной работы был реализован минимальный серверный проект со следующей структурой:

index.js — основной файл Express-сервера с описанием маршрутов API

data.json — файл для временного хранения данных (в памяти через JSON)

swagger.json — статическое описание API в формате OpenAPI

.gitignore — файл исключений Git

Данная структура является упрощённой и соответствует минимальным требованиям лабораторной работы.

### 2. Реализованные элементы

В ходе выполнения лабораторной работы были реализованы следующие компоненты:

Express-сервер с базовой конфигурацией

Middleware:

express.json() — для обработки JSON-тела запросов

cors — для поддержки CORS

morgan — для логирования HTTP-запросов

CRUD операции для ресурса tasks:

GET /tasks — получение списка задач

GET /tasks/:id — получение задачи по идентификатору

POST /tasks — создание новой задачи

DELETE /tasks/:id — удаление задачи

Валидация входных данных:

поле title — обязательное, длина 1–100 символов

поле done — логическое значение

поле dueDate — строка в формате ISO-даты

Хранение данных во внешнем JSON-файле без использования базы данных

Обработка ошибок:

возврат корректных HTTP статус-кодов (400, 404, 422, 201, 204)

сообщения об ошибках в формате JSON

Swagger UI документация, доступная по адресу /docs

Статическое описание API в файле swagger.json

## Инструкция по запуску

### Установка зависимостей

```bash
npm install  
```

### Запуск сервера

**Режим разработки:**

```bash
npm run dev
```

**Обычный запуск:**

```bash
npm start
```

Сервер запустится на порту 3000 (или указанном в `.env`).

### Доступ к документации

После запуска сервера документация Swagger доступна по адресу:

```text
http://localhost:3000/docs
```

### Примеры запросов

**Получить списка задач:**

```bash
GET /tasks
```

**Получить задач по ID:**

```bash
GET /tasks/1
```

**Создать новый опрос:**

```bash
POST /tasks
Content-Type: application/json

{
  "title": "Подготовить лабораторную работу",
  "done": false,
  "dueDate": "2025-12-31"
}
```

**Удалить опрос:**

```bash
DELETE /tasks/1
```
