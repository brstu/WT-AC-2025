## Цель работы

Освоить работу с DOM: добавление, удаление и изменение элементов интерфейса. Научиться обрабатывать события, использовать делегирование, работать с клавиатурной доступностью. Реализовать клиентскую валидацию форм.

### Вариант №26

Тема: Музыкальная медиатека
Табы жанров и плейлистов, модальное окно «Играет сейчас», форма поиска и форма обратной связи.

## Ход выполнения работы

### 1. Структура проекта

```
\---src
    |   index.html
    |   styles.css
    |   main.js
    |
    \---img
            exercise1.jpg
            exercise2.jpg
            exercise3.jpg
            exercise4.jpg
            exercise5.jpg
```

- `index.html` — основная страница с дневником тренировок
- `styles.css` — стилизация и медиазапросы
- `main.js` — логика интерактивных компонентов
- `img/` — изображения упражнений

### 2. Реализованные элементы

    В ходе выполнения лабораторной работы реализованы следующие функциональные элементы:
    Семантическая HTML5-разметка (header, nav, main, section, article, footer)
    Табы для переключения между жанрами и плейлистами
    Модальное окно «Играет сейчас» с информацией о выбранном треке
    Аккордеон с советами по прослушиванию музыки и фокусировке
    Форма поиска треков по названию и исполнителю
    Форма обратной связи с клиентской валидацией:
    имя — обязательное поле
    e-mail — проверка корректности формата
    сообщение — минимум 20 символов
    Кнопка отправки формы неактивна при невалидных данных
    Делегирование событий на контейнере списка треков:
    воспроизведение
    лайк
    удаление трека
    Управление с клавиатуры (Tab, Enter, Space, Escape)
    Использование ARIA-атрибутов для табов, аккордеона  и модального окна
    Адаптивная вёрстка (mobile-first, брейкпоинты 601px и 1025px)
    Сохранение состояния (лайки, фильтры, активные табы) в localStorage
