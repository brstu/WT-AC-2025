# Лабораторная работа — Вариант 27: «Каталог студенческих кружков» (SPA + CRUD + заявки)

## 1) Описание
В рамках лабораторной работы реализовано одностраничное приложение (SPA) **«Каталог студенческих кружков»** на **чистом JavaScript (ES6 modules)** без библиотек и фреймворков.  
Приложение использует **hash-маршрутизацию** и работает с **локальным mock API** на базе **json-server** (`db.json`). Реализованы требования задания:

- маршруты (hash): `#/items`, `#/items/:id`, `#/new`, `#/items/:id/edit`
- состояния UI на экранах: **loading / error / empty**
- CRUD: `GET` список/деталь, `POST` создание, `PUT` обновление, `DELETE` удаление
- UX: **подтверждение удаления**, **блокировка submit** во время отправки, **валидация форм**, **уведомления** об успехе/ошибках
- сохранение параметров поиска в **hash** (`#/items?q=...&category=...`)
- архитектура по модулям: `router`, `api`, `views` (+ `ui`, `utils`)
- адаптивная верстка списка и форм

Дополнительно реализована **форма заявки** на вступление в кружок на детальной странице (создание `applications` через API) и вывод последних заявок.

## 3) Параметры варианта
- Тема: **Каталог студенческих кружков**
- Источник данных: **локальный mock API** (json-server)
  - файл данных: `db.json`
  - коллекции: `clubs`, `applications`
- Сущность `clubs` (кружки):
  - поля: `id`, `name`, `category`, `description`, `meeting`, `contactEmail`, `createdAt`
- Сущность `applications` (заявки):
  - поля: `id`, `clubId`, `studentName`, `email`, `course`, `message`, `createdAt`

## 4) Маршруты (hash)
- `#/items` — список кружков + поиск/фильтр (параметры сохраняются в hash)
- `#/items/:id` — детальная страница кружка + форма заявки + последние заявки
- `#/new` — создание кружка
- `#/items/:id/edit` — редактирование кружка

## 5) Функциональность (CRUD / UX)
### CRUD кружков (clubs)
- `GET /clubs?...` — получение списка (поиск `q`, фильтр `category`)
- `GET /clubs/:id` — получение детали
- `POST /clubs` — создание кружка
- `PUT /clubs/:id` — обновление кружка (требование PATCH/PUT выполнено через PUT)
- `DELETE /clubs/:id` — удаление кружка

### Заявки (applications)
- `POST /applications` — отправка заявки на вступление (привязка к `clubId`)
- `GET /applications?clubId=:id&_sort=createdAt&_order=desc&_limit=5` — последние заявки по кружку

### UX и валидация
- удаление подтверждается через `confirm()`
- кнопка submit блокируется во время отправки (защита от повторной отправки)
- валидация форм:
  - кружок: обязательные поля, минимальная длина описания
  - заявка: обязательные поля, проверка email, минимальная длина сообщения
- уведомления (toast): успех/ошибка операций CRUD и отправки заявки
- состояния UI: `loading` (skeleton/плашка), `error` (сообщение + retry), `empty` (пустой список + CTA)

## 6) Минимальные технические требования

- адаптивная верстка: сетка карточек и формы корректно работают на разных ширинах
