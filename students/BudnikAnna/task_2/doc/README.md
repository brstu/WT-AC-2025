# Лабораторная работа — Вариант 27: «Коллекция настолок»

## 1) Описание
В рамках лабораторной работы реализован одностраничный интерфейс «Коллекция настольных игр» на чистом JavaScript (ES6+), без библиотек.  
Приложение поддерживает:
- **Табы жанров** для фильтрации списка игр
- **Модальное окно** с правилами выбранной игры
- **Делегирование событий** для действий на карточках (лайк / открыть правила)
- **Форму обратной связи** с валидацией при вводе и при отправке
- **Доступность (a11y)**: управление с клавиатуры, ARIA-атрибуты, focus trap в модалке

Изображения реализованы как **локальные SVG** в папке `assets/` (без внешних ссылок).

## 5) Реализованные интерактивные компоненты (минимум 3)
1. **Tabs (табы жанров)**  
   - Роли/ARIA: `role="tablist"`, `role="tab"`, `role="tabpanel"`, `aria-selected`, `aria-controls`, `aria-labelledby`
   - Клавиатура: `←/→`, `Home/End`, `Enter/Space` для выбора

2. **Modal (модальное окно правил)**  
   - `role="dialog"`, `aria-modal="true"`, `aria-labelledby`
   - Закрытие по `Esc`, по кнопке и по клику на overlay
   - Реализован **focus trap** (фокус не выходит за модалку)

3. **Accordion (FAQ)**  
   - `aria-expanded`, `aria-controls`, `role="region"`, `aria-labelledby`
   - Управление с клавиатуры через стандартную кнопку (Tab/Enter/Space)

## 6) Форма с валидацией
Форма: `#feedback-form`

Поля:
- **Имя** — обязательное, минимум 2 символа
- **E-mail** — необязательное, но если заполнено → должен быть валидный формат
- **Сообщение** — минимум 20 символов

Требования выполнены:
- Валидация **при вводе** и **при отправке**
- Пользовательские сообщения ошибок (под полями)
- Кнопка `Отправить` **неактивна**, если форма невалидна
- Отправка **не уходит на сервер**: `preventDefault()`, результат выводится в блок `#form-result`

## 7) Делегирование событий
Выполнено делегирование на контейнерах:
- Контейнер карточек (внутри панели таба): один обработчик на `#genre-panels`
  - `data-action="toggle-like"` — лайк/анлайк
  - `data-action="open-rules"` — открыть модалку правил
- Дополнительно:
  - делегирование закрытия модалки по `data-action="close-modal"`
  - делегирование для аккордеона FAQ

## 8) Доступность (a11y)
Реализовано:
- Навигация с клавиатуры: `Tab/Shift+Tab`, `Enter/Space`, `Esc`
- ARIA-атрибуты для табов/модалки/аккордеона
- `aria-live` для сообщений об ошибках формы и блока результата
- **Skip-link** (“Перейти к содержимому”) для быстрого перехода к `main`
- Управление фокусом при открытии/закрытии модалки (возврат фокуса на кнопку-открывалку)
