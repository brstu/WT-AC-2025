## Цель работы

Создание REST API для ресурса на Express с валидацией входных данных, централизованной обработкой ошибок и документацией Swagger. Реализовать базовую структуру Express‑приложения и CRUD эндпоинты, добавить валидацию (joi), обработчик ошибок и документацию OpenAPI/Swagger UI.

### 2. Реализованные элементы
**Основная функциональность:**

- Express-сервер с middleware (json, cors, morgan)
- REST API для управления местами путешествий
- CRUD операции для ресурса "places"
- Вложенный ресурс "reviews" для отзывов к местам
- Валидация данных с помощью Joi
- Документация API через Swagger UI
- Переменные окружения в .env файле

**Реализованные эндпоинты:**

- `GET /places` - получение списка мест (с параметрами q, limit, offset)
- `GET /places/:id` - получение места по ID
- `POST /places` - создание нового места
- `PUT /places/:id` - обновление места
- `DELETE /places/:id` - удаление места
- `GET /places/:placeId/reviews` - получение отзывов места
- `POST /places/:placeId/reviews` - создание отзыва
- `DELETE /reviews/:id` - удаление отзыва

**Валидация:**

- Проверка длины названия места (1-100 символов)
- Проверка описания (1-500 символов)
- Проверка рейтинга (0-5)
- Проверка обязательных полей

### 3. Установка и запуск

```bash
cd src
npm install
npm start
```

Сервер запустится на порту 3000. Документация доступна по адресу: <http://localhost:3000/docs>
