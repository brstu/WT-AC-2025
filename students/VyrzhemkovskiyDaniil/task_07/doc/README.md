```
src/
```

---

### 2. Основные технологии и библиотеки

- **React 18** + **Vite** — сборка и рендеринг приложения
- **React Router v6** — маршрутизация между страницами
- **Redux Toolkit** + **RTK Query** — управление состоянием и работа с API
- **React Hook Form** — управление формами и валидация
- **Zod** — схема валидации для форм
- **CSS Modules** — стилизация компонентов

---

### 3. Функциональность приложения

#### 3.1. Страницы

1. **Главная страница** (`/`) — отображение списка всех фильмов с поиском и фильтрацией
2. **Детальная страница фильма** (`/movies/:id`) — полная информация о конкретном фильме
3. **Создание фильма** (`/movies/new`) — форма добавления нового фильма в коллекцию
4. **Редактирование фильма** (`/movies/:id/edit`) — форма изменения данных фильма
5. **Страница 404** (`/*`) — обработка несуществующих маршрутов

#### 3.2. Маршрутизация

- Вложенные маршруты с общим Layout
- Динамические параметры (`:id`) для детальных страниц
- Навигация через `<Link>` и программная навигация через `useNavigate()`
- Обработка несуществующих маршрутов

#### 3.3. Работа с данными (CRUD)

- **GET /movies** — получение списка всех фильмов
- **GET /movies/:id** — получение детальной информации о фильме
- **POST /movies** — создание нового фильма
- **PUT /movies/:id** — обновление существующего фильма
- **DELETE /movies/:id** — удаление фильма из коллекции

#### 3.4. Состояния интерфейса

- **Загрузка** (`loading`) — индикатор при выполнении запросов
- **Ошибка** (`error`) — отображение ошибок при неудачных запросах
- **Пустое состояние** (`empty`) — сообщение при отсутствии фильмов
- **Уведомления** — всплывающие сообщения об успешных операциях

#### 3.5. Формы и валидация

- Валидация всех обязательных полей
- Проверка типов данных (числа, строки)
- Ограничения по значениям (год, рейтинг, длительность)
- Информативные сообщения об ошибках

### 4. Запуск проекта

#### 4.1. Установка зависимостей

Откройте проект в терминале и выполните:

```bash
npm install
```

#### 4.2. Настройка окружения

Создайте файл `.env` в корне проекта со следующим содержимым:

```env
VITE_API_URL=http://localhost:3000
```

#### 4.3. Запуск в режиме разработки

Выполните команду для запуска сервера разработки:

```bash
npm run dev
```

#### 4.4. Доступ к приложению

После успешного запуска откройте браузер и перейдите по адресу:

```
http://localhost:5173
```

#### 4.5. Использование мокового API

Приложение использует встроенный моковый API, который не требует дополнительной настройки. Все данные хранятся в памяти браузера.

## Вывод

В ходе выполнения лабораторной работы было создано полноценное одностраничное приложение (SPA) на React для управления коллекцией фильмов кинофестиваля. Реализованы все основные требования задания:

1. **Маршрутизация** — навигация между 5 страницами с помощью React Router v6, включая динамические маршруты для детальных страниц и обработку 404 ошибок.

2. **Работа с данными** — полный CRUD функционал через RTK Query с поддержкой состояний загрузки, ошибок и пустых состояний.

3. **Формы с валидацией** — интуитивные формы создания и редактирования фильмов с комплексной валидацией через React Hook Form и Zod.

4. **Управление состоянием** — централизованное состояние через Redux Toolkit с оптимистичными обновлениями и автоматическим кэшированием.

5. **Пользовательский интерфейс** — современный адаптивный дизайн с тёмной темой, анимациями и уведомлениями.
