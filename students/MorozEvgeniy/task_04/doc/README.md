### Вариант №15: Справочник гаджетов (Список, Деталь, Форма добавления)

## Ход выполнения работы

### 1. Структура проекта

Проект реализован в соответствии с требованиями модульности и использует следующую структуру:

- `index.html` — основная страница приложения.
- `style.css` — стилизация, включая адаптивную вёрстку и стили для состояний.
- `js/main.js` — точка входа, инициализация роутера и маршрутов.
- `js/router.js` — hash-маршрутизатор с поддержкой динамических параметров (ID) и query-параметров (поиск).
- `js/api.js` — Mock API на основе `localStorage` для эмуляции CRUD-операций и сохранения данных.
- `js/utils.js` — вспомогательные функции (уведомления, лоадеры).
- `js/views/list.js` — отображение списка гаджетов с полем для поиска.
- `js/views/detail.js` — отображение деталей конкретного гаджета.
- `js/views/form.js` — универсальная форма для создания и редактирования гаджетов.

---

### 2. Описание маршрутов

Маршрутизация построена на основе **hash** и событий `hashchange`/`load`.

| Маршрут | Описание |
|---------|----------|
| `#/items` | Список всех гаджетов (поддерживает поиск: `#/items?q=...`) |
| `#/items/:id` | Детальная информация о конкретном гаджете |
| `#/items/:id/edit` | Форма для редактирования существующего гаджета |
| `#/new` | Форма для добавления нового гаджета |

---

### 2. Описание API

Приложение использует Mock REST API, который имитирует задержку сети и полностью поддерживает CRUD-операции для сущности "гаджет".

| Метод  | URL                   | Описание |
|--------|----------------------|----------|
| GET    | `/gadgets`            | Получение списка гаджетов (поддерживает фильтрацию по имени) |
| GET    | `/gadgets/:id`        | Получение одного гаджета по id |
| POST   | `/gadgets`            | Создание нового гаджета (данные из формы) |
| UPDATE | `/gadgets/:id`        | Обновление гаджета по id |
| DELETE | `/gadgets/:id`        | Удаление гаджета по id |

---

### 3. Запуск проекта

#### 3.1 Локальный сервер (Live Server)

* Открыть проект в VSCode.
* Запустить Live Server.
* Сайт откроется по адресу: `http://127.0.0.1:5500/` (или аналогичный).

Приложение использует **localStorage** для хранения данных, поэтому Mock-сервер (`json-server`) не требуется.
