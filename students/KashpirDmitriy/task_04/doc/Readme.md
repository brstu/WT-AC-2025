## Цель работы

* Реализовать простой SPA без фреймворков с маршрутизацией и состояниями загрузки/ошибок.
* Выполнить CRUD к учебному API (реальному или mock).

---

### Вариант №34 — Галерея портфолио дизайнеров

## Ход выполнения работы

### 1. Структура проекта

* `index.html` — основная страница приложения.
* `css/styles.css` — стилизация интерфейса.
* `js/main.js` — логика на чистом JS, инициализация роутера и маршрутов.
* `js/router.js` — простой hash-маршрутизатор для SPA.
* `js/api.js` — функции для работы с REST API (CRUD работ дизайнеров).
* `js/views/workDetail.js` — отображение деталей работы с формой связи.
* `js/views/workForm.js` — форма для добавления и редактирования работы.
* `js/views/worksList.js` — отображение списка работ с поиском и фильтрацией.
* `db.json` — база данных с портфолио работ дизайнеров.

---

### 2. Описание маршрутов

| Маршрут            | Описание                                               |
|--------------------|--------------------------------------------------------|
| `#/works`          | Список всех работ дизайнеров                           |
| `#/works/:id`      | Деталь работы с информацией о дизайнере и формой связи |
| `#/works/:id/edit` | Редактирование существующей работы                     |
| `#/new`            | Форма для добавления новой работы                      |

---

### 3. Описание API

Приложение использует REST API с поддержкой CRUD для работ дизайнеров.

| Метод  | URL                   | Описание                                                     |
|--------|---------------------- |--------------------------------------------------------------|
| GET    | `/works`              | Получение списка работ (опционально с поиском: `?q=<поиск>`) |
| GET    | `/works/:id`          | Получение одной работы по id                                 |
| POST   | `/works`              | Создание новой работы (JSON в теле запроса)                  |
| PATCH  | `/works/:id`          | Редактирование работы по id (JSON в теле запроса)            |
| DELETE | `/works/:id`          | Удаление работы по id                                        |

---

### 4. Запуск проекта

#### 4.1 Локальный сервер (Live Server)

* Открть проект в VSCode.
* Запустить Live Server.
* Сайт откроется по адресу:

```bash
http://127.0.0.1:5500/lab4/
```

При первом запуске хэш автоматически установится в #/works.

---

#### 4.2 Mock‑сервер (json-server)

Для работы API без настоящей базы данных:

* Установить json-server (если ещё не установлен):

```bash
npm install -g json-server
```

* Запустить сервер:

```bash
json-server --watch db.json --port 3000
```

* Теперь API доступно по:

```bash
http://localhost:3000/works
```

Все CRUD-запросы работают через этот mock-сервер.

## Вывод

Была выполнена лабораторная работа по созданию простой SPA без фреймворков — Галереи портфолио дизайнеров. Приложение позволяет просматривать портфолио работ различных дизайнеров, переходить между страницами через hash-маршрутизацию и показывает состояния загрузки, ошибки и пустого списка. Реализованы все операции CRUD: просмотр списка и деталей работ, добавление, редактирование и удаление портфолио. На странице деталей каждой работы находится форма связи для заинтересованных клиентов. Для работы использовался mock‑сервер json-server. Все функции проверялись через интерфейс и DevTools.
