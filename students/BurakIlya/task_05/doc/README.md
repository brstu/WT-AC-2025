## Цель работы

Создание REST API для ресурса на Express с валидацией входных данных, централизованной обработкой ошибок и документацией Swagger. Реализация базовой структуры Express‑приложения и CRUD эндпоинты.

---

### Вариант №29

**Тема:** API коллекций фотографий

## Ход выполнения работы

### 1. Структура проекта

```text
src/
├── server.js       — основной файл сервера
├── data.json       — хранение данных
├── package.json    — зависимости проекта
├── .env           — переменные окружения
└── .gitignore     — игнорируемые файлы
```

### 2. Реализованные элементы

- REST API для управления коллекциями фотографий
- CRUD операции (GET, POST, PUT, PATCH, DELETE)
- Middleware: express.json(), cors, morgan
- Хранение данных в JSON файле
- Базовая валидация входных данных
- Swagger документация на `/docs`
- Параметры запросов: q (поиск), limit, offset
- Обработка ошибок

### 3. Эндпоинты API

**GET /photos** — получить список фотографий  
Параметры: `q` (поиск), `limit`, `offset`

**GET /photos/:id** — получить фотографию по ID

**POST /photos** — создать новую фотографию  
Тело запроса: `{ title, description, url, tags }`

**PUT /photos/:id** — обновить фотографию

**PATCH /photos/:id** — частично обновить фотографию

**DELETE /photos/:id** — удалить фотографию

### 4. Запуск проекта

1. Установить зависимости:

```bash
cd src
npm install
```

1. Запустить сервер:

```bash
npm start
```

или для разработки:

```bash
npm run dev
```

1. API доступно на `http://localhost:3000`
1. Документация Swagger: `http://localhost:3000/docs`
